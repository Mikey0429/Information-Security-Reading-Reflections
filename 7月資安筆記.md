打開一個網站時DNS會先找到IP->http會與伺服器對話請求->伺服器回應html/css/js/圖片等等的資料->最後瀏覽器整合起來
另外還有一些元件可以提供額外的功能也能幫助網站更有效率的去運作
（負載平衡器）
用來確保高流量的網站能承受負載
在伺服器失去回應的時候可以提供故障轉移
他可以提升網站/伺服器的可用性
（CDN）
這就是負責將網站上面的html、css、js、圖片等等的資訊託管到伺服器上
只有要使用時會請求最近的伺服器做回應
（資料庫）
最近為了管理學生資料和健身課程預約時間請ai做了一個網站就有接觸到資料庫
他是網站要存放使用者資料的地方
（WAF）web防火牆
類似網站的保鑣
他坐在你的請求和伺服器的中間
他會檢查請求像不像駭客攻擊手法
判斷是真人還是機器人
速率限制避免暴力攻擊
只要有可疑的請求他就會擋掉不給他進入伺服器
剛剛這些都提到了伺服器
網頁伺服器是一個監聽連線+送檔案的軟體
他會監聽看瀏覽器是否有與他做連線請求
而瀏覽器請求了什麼伺服器就會用http把網頁的內容送出去
伺服器送到檔案都是在一個指定的資料夾（世界上某一台專門電腦的根目錄）
而這裡linux和win的位置都有不同
在傳送內容的部分也有分靜態和動態
靜態就是一些永遠不會改變的內容，圖片、js、css等等
而動態就是會隨著請求的不同而改變的
例如：搜尋頁面、建立文章等等
動態內容需要後端去做運算，透過腳本或是程式語言去完成
而後端無法在檢視html中就看到他的運作
Html只看的到他運作後的結果

7/3 OverTheWire Natas natas0
題目:  
You can find the password for the next level on this page. 
您可以在此頁面找到下一個關卡的密碼。

看到一個網頁第一件事就是打開他的html查看有什麼訊息資料
有兩種方式
1. F12 (裡面有各種資料)
    1. 缺點 :檢測器屬於動態無法直接搜尋到<!—字符 他只能搜尋內容
    2. 以這關為例:解題要改搜尋關鍵字 password或 複製題目
2. Ctrl + U (原始檔藍圖)
    1. 純粹的文字檔原始碼藍圖 (最原始)
    2. 因為是純文字檔所以可以搜尋<!—
    3. 缺點是網頁不會更動,完全「唯讀」無法互動
    ***
    通常Ctrl + U 保留最初的痕跡
    開發者寫死的密碼或註解 通常都藏在這裡
    ***

    **衍伸搜尋**
    這關內容比對資安事故 (前端資料洩漏)
    漏洞名稱: 資訊洩漏（Information Exposure / Information Disclosure）
    漏洞成因: 
    開發者以為只要畫面上看不到,使用者就不會找到  
    使用者像伺服器請求網頁後,網頁的所有HTML、JS、CSS檔案,都會完整的下載進使用者的瀏覽器中,使用者可以幾乎100%利用
    危險資訊: (AI 資訊)    
      1. AWS / Google Cloud 的 API 存取金鑰
      2. 內部測試伺服器的 IP 位置與路徑
      3. 舊版的被註解掉的程式碼
    防禦: 
      1. 永遠不要將機密資訊寫在前端程式碼中
      2. 導入自動化工具 打包工具在部屬到正式環境前 ,會自動把所有HTML、JS註解移除,避免人為疏忽
    **資安思維總結：**
     做到零信任。使用者可以看光、修改所有前端的東西。
