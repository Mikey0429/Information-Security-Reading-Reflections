7/3 OverTheWire Natas natas0
題目:  
You can find the password for the next level on this page. 
您可以在此頁面找到下一個關卡的密碼。

看到一個網頁第一件事就是打開他的html查看有什麼訊息資料
有兩種方式
1. F12 (裡面有各種資料)
    1. 缺點 :檢測器屬於動態無法直接搜尋到<!—字符 他只能搜尋內容
    2. 以這關為例:解題要改搜尋關鍵字 password或 複製題目
2. Ctrl + U (原始檔藍圖)
    1. 純粹的文字檔原始碼藍圖 (最原始)
    2. 因為是純文字檔所以可以搜尋<!—
    3. 缺點是網頁不會更動,完全「唯讀」無法互動
    ***
    通常Ctrl + U 保留最初的痕跡
    開發者寫死的密碼或註解 通常都藏在這裡
    ***

    **衍伸搜尋**
    這關內容比對資安事故 (前端資料洩漏)
    漏洞名稱: 資訊洩漏（Information Exposure / Information Disclosure）
    漏洞成因: 
    開發者以為只要畫面上看不到,使用者就不會找到  
    使用者像伺服器請求網頁後,網頁的所有HTML、JS、CSS檔案,都會完整的下載進使用者的瀏覽器中,使用者可以幾乎100%利用
    危險資訊: (AI 資訊)    
      1. AWS / Google Cloud 的 API 存取金鑰
      2. 內部測試伺服器的 IP 位置與路徑
      3. 舊版的被註解掉的程式碼
    防禦: 
      1. 永遠不要將機密資訊寫在前端程式碼中
      2. 導入自動化工具 打包工具在部屬到正式環境前 ,會自動把所有HTML、JS註解移除,避免人為疏忽
    **資安思維總結：**
     做到零信任。使用者可以看光、修改所有前端的東西。

- AI延伸閱讀筆記
    漏洞名稱:
        目錄列舉

    漏洞起因: 
        網路伺服器的內建功能中有”自動索引”和”目錄列舉”功能
        使用者發送請求看一個資料夾時,伺服器會優先尋找有沒有
        `index.html`、`index.php` 或 `default.html`
        這種預設首頁檔
        當伺服器找不到”首頁檔”,又剛好沒有關閉Autoindex（自動索引）功能
        它會自動掃描資料夾內的"檔案"和"子資料夾”
        然後會自動生成一個HTML網頁把這些檔案變成超連結顯示在瀏覽器上
        (就像這次的題目這樣)

     解決方式:
        1. 修改伺服器設定檔
             a. 直接告訴web server "不管遇到甚麼狀況都不能啟動自動索引”
        2. 放置空白"首頁檔”
             b. 讓伺服器優先找到”空白首頁檔”
        3. 打從一開始就不該把敏感資料放在”網頁公開目錄”

但畢竟重複做有時候會膩,所以我又開始想:是不是也能用現在的遠端虛擬機玩玩看 Natas 這個 Web 主題?因為這樣,我開始慢慢地解第 1 關到第 6 關。剛開始都是看不懂關卡,所以做得比較慢。

學到的 curl 用法:
用 -s 讓版面變乾淨一點,畢竟用手機看,如果亂亂的很難抓重點
用 | grep 直接過濾出想看的資訊
用 -u 帶入每一關的帳號和密碼
用 -e 偽造 HTTP Referer 標頭(第 4 關)
用 -b 送 Cookie 來偽造登入狀態(第 5 關)

我用 export 把每一關的密碼儲存到一個 .env 檔案裡,這樣不用每次都要重找密碼,只需要呼叫變數 $PW* 就能取到對應的密碼:
curl -u natas*:PW* http://natas*.natas.labs.overthewire.org
後面發現手機很常打錯字,所以設定了不少 export。
有時候會忘記用 source 重新載入設定檔,結果變數是空的,curl 打的網址就變成沒展開的樣子,伺服器直接回 404。

有時候亂猜答案也會跳出 403 或 401。不過之前學過狀態碼,現在看到大概就能反推是哪裡出問題:
401 是帳密沒帶對
403 是路徑存在但沒權限
404 是網址根本不存在

因為終端機只有文字,發現很多資訊都直接浮現出來。前 6 關幾乎都是把密碼藏在明文或 HTML 裡,只要改標頭、改 Cookie,或找一下有沒有其他路徑,就能發現密碼。
